DSA Interview Revision Guide

1. Regular Expression Matching Hard

DP Recursion + Memoization
Intuition: * means zero or more of the preceding element, not a wildcard. Two choices when you see x*: skip it entirely (zero occurrences → j+2) or consume one character if it matches (stay at j). This branching makes it a DP problem. 
class Solution {
    Boolean[][] memo;

    public boolean isMatch(String s, String p) {
        memo = new Boolean[s.length() + 1][p.length() + 1];
        return solve(s, p, 0, 0);
    }

    private boolean solve(String s, String p, int i, int j) {
        if (j == p.length()) return i == s.length();
        if (memo[i][j] != null) return memo[i][j];

        boolean firstMatch = (i < s.length()) &&
            (s.charAt(i) == p.charAt(j) || p.charAt(j) == '.');

        boolean result;
        if (j + 1 < p.length() && p.charAt(j + 1) == '*') {
            result = solve(s, p, i, j + 2) ||              // skip x*
                     (firstMatch && solve(s, p, i + 1, j)); // consume one
        } else {
            result = firstMatch && solve(s, p, i + 1, j + 1);
        }
        memo[i][j] = result;
        return result;
    }
}
Time: O(n × m)  |  Space: O(n × m)

2. kSum (3Sum / 4Sum Generalized)

Recursion Backtracking Two Pointers
Intuition: kSum fixes one element, calls (k-1)Sum recursively. Base case is 2Sum. current is a shared list → must backtrack. Use long for target to avoid overflow. 
class Solution {
    List<List<Integer>> result = new ArrayList<>();

    public List<List<Integer>> fourSum(int[] nums, int target) {
        Arrays.sort(nums);
        solve(nums, (long) target, 4, 0, new ArrayList<>());
        return result;
    }

    private void solve(int[] nums, long target, int k, int start, List<Integer> current) {
        if (k == 2) { twoSum(nums, target, start, current); return; }
        for (int i = start; i < nums.length; i++) {
            if (i > start && nums[i] == nums[i - 1]) continue;
            current.add(nums[i]);
            solve(nums, target - nums[i], k - 1, i + 1, current);
            current.remove(current.size() - 1); // backtrack
        }
    }

    private void twoSum(int[] nums, long target, int start, List<Integer> current) {
        int left = start, right = nums.length - 1;
        while (left < right) {
            long sum = (long) nums[left] + nums[right];
            if (sum == target) {
                List<Integer> combo = new ArrayList<>(current);
                combo.add(nums[left]); combo.add(nums[right]);
                result.add(combo);
                left++; right--;
                while (left < right && nums[left] == nums[left-1]) left++;
                while (left < right && nums[right] == nums[right+1]) right--;
            } else if (sum < target) left++;
            else right--;
        }
    }
}
Time: O(n^(k-1)) — e.g. 3Sum=O(n²), 4Sum=O(n³)  |  Space: O(k)

3. Combination Sum I Medium

Backtracking Unbounded
Intuition: Pick current element (stay at same index — reuse allowed) or skip it (move to next). Check success BEFORE pruning.
Bug to avoid: Checking candidates[index] > target BEFORE target == 0 causes valid answers to be skipped. 
class Solution {
    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        List<List<Integer>> combos = new ArrayList<>();
        Arrays.sort(candidates);
        solve(candidates, target, 0, combos, new ArrayList<>());
        return combos;
    }
    private void solve(int[] c, int target, int idx,
                       List<List<Integer>> combos, List<Integer> cur) {
        if (target == 0) { combos.add(new ArrayList<>(cur)); return; }
        if (target < 0 || idx >= c.length) return;
        if (c[idx] > target) return;

        cur.add(c[idx]);
        solve(c, target - c[idx], idx, combos, cur); // pick (stay)
        cur.remove(cur.size() - 1);
        solve(c, target, idx + 1, combos, cur);      // skip
    }
}
Time: O(n^(target/min))  |  Space: O(target/min)

4. Combination Sum II Medium

Backtracking Bounded Skip Duplicates
Intuition: Same pick/skip but pass index+1 when picking (can't reuse). Sort array and skip duplicates in the skip branch. 
class Solution {
    public List<List<Integer>> combinationSum2(int[] c, int target) {
        List<List<Integer>> combos = new ArrayList<>();
        Arrays.sort(c);
        solve(c, target, 0, combos, new ArrayList<>());
        return combos;
    }
    private void solve(int[] c, int target, int idx,
                       List<List<Integer>> combos, List<Integer> cur) {
        if (target == 0) { combos.add(new ArrayList<>(cur)); return; }
        if (target < 0 || idx >= c.length) return;

        cur.add(c[idx]);
        solve(c, target - c[idx], idx + 1, combos, cur); // pick (move on)
        cur.remove(cur.size() - 1);

        while (idx+1 < c.length && c[idx] == c[idx+1]) idx++; // skip dupes
        solve(c, target, idx + 1, combos, cur);
    }
}
Time: O(2^n)  |  Space: O(n)

5. Reverse Integer Medium

Math Overflow Handling
Intuition: Pop last digit with % 10, build result with res * 10 + digit. Check overflow BEFORE the multiply using MAX_VALUE / 10. Java's % and / handle negatives naturally.
Bugs to avoid: (1) while(x > 0) fails for negatives — use x != 0. (2) Math.abs() inside loop breaks negative handling. (3) Must check overflow before res*10, not after. 
class Solution {
    public int reverse(int x) {
        int res = 0;
        while (x != 0) {
            int digit = x % 10;
            if (res > Integer.MAX_VALUE/10 ||
                (res == Integer.MAX_VALUE/10 && digit > 7)) return 0;
            if (res < Integer.MIN_VALUE/10 ||
                (res == Integer.MIN_VALUE/10 && digit < -8)) return 0;
            res = res * 10 + digit;
            x /= 10;
        }
        return res;
    }
}
Time: O(log x)  |  Space: O(1)

6. Count Subarrays With Median K

Prefix Sum HashMap
Intuition: Transform: >k → +1, <k → -1, ==k → 0. For k to be median, subarray sum = 0 (odd) or 1 (even). Store left prefix sum frequencies in HashMap, match with right extensions.
Bugs to avoid: (1) Use HashMap not HashSet — counting problem needs frequencies. (2) Use separate += not if(a || b) — || short-circuits and misses valid subarrays. 
class Solution {
    public int countSubarrays(int[] nums, int k) {
        int n = nums.length, m = 0;
        for (int i = 0; i < n; i++)
            if (nums[i] == k) { m = i; break; }

        Map<Integer,Integer> left = new HashMap<>();
        left.put(0, 1);
        int sum = 0;
        for (int i = m-1; i >= 0; i--) {
            sum += (nums[i] > k) ? 1 : -1;
            left.merge(sum, 1, Integer::sum);
        }
        int res = 0; sum = 0;
        for (int i = m; i < n; i++) {
            sum += (nums[i] > k) ? 1 : (nums[i] < k) ? -1 : 0;
            res += left.getOrDefault(-sum, 0);
            res += left.getOrDefault(-sum + 1, 0);
        }
        return res;
    }
}
Time: O(n)  |  Space: O(n)

7. Sell Colored Balls (Max Profit)

Binary Search Arithmetic Series Greedy
Intuition: Instead of selling one-by-one (TLE), binary search for threshold T. Sell all balls above T using arithmetic series. Handle leftover orders at value T. Think of balls as stacked columns — sell entire rows from top down. 
class Solution {
    public int maxProfit(int[] inv, int orders) {
        long MOD = 1_000_000_007;
        long lo = 0, hi = 0;
        for (int v : inv) hi = Math.max(hi, v);

        while (lo < hi) {
            long mid = lo + (hi - lo) / 2;
            long cnt = 0;
            for (int v : inv) if (v > mid) cnt += v - mid;
            if (cnt <= orders) hi = mid;
            else lo = mid + 1;
        }
        long T = lo, profit = 0, rem = orders;
        for (int v : inv) {
            if (v > T) {
                profit = (profit + (long)v*(v+1)/2 - T*(T+1)/2) % MOD;
                rem -= v - T;
            }
        }
        profit = (profit + rem * T) % MOD;
        return (int) profit;
    }
}
Time: O(n × log(max_val))  |  Space: O(1)

8. Longest Palindromic Substring Medium

DP Gap-based
Intuition: dp[i][j] = is substring i..j a palindrome? Iterate by gap. gap=0: true. gap=1: true if chars match. gap≥2: chars match AND dp[i+1][j-1].
Bug to avoid: gap=1 means TWO characters — it's NOT always true, only when s[i]==s[j]. 
class Solution {
    public String longestPalindrome(String s) {
        int n = s.length();
        boolean[][] dp = new boolean[n][n];
        int[] idx = new int[2];
        for (int gap = 0; gap < n; gap++) {
            for (int i = 0, j = gap; j < n; i++, j++) {
                if (gap == 0) dp[i][j] = true;
                else if (gap == 1) dp[i][j] = s.charAt(i) == s.charAt(j);
                else dp[i][j] = s.charAt(i) == s.charAt(j) && dp[i+1][j-1];
                if (dp[i][j]) { idx[0] = i; idx[1] = j; }
            }
        }
        return s.substring(idx[0], idx[1] + 1);
    }
}
Time: O(n²)  |  Space: O(n²)

9. Search in Rotated Sorted Array Medium

Binary Search
Intuition: Find min element index first, then binary search in the correct half.
Bugs to avoid: (1) mid = low + (high-low)/2 not high - (high-low)/2. (2) low = mid+1 not low = mid — causes infinite loop. 
class Solution {
    public int search(int[] nums, int target) {
        int n = nums.length;
        int min = getMinIndex(nums, n);
        int lo = target <= nums[n-1] ? min : 0;
        int hi = target > nums[n-1] ? min - 1 : n - 1;
        while (lo <= hi) {
            int mid = lo + (hi - lo) / 2;
            if (nums[mid] == target) return mid;
            if (nums[mid] < target) lo = mid + 1;
            else hi = mid - 1;
        }
        return -1;
    }
    private int getMinIndex(int[] a, int n) {
        int lo = 0, hi = n - 1;
        while (lo < hi) {
            int mid = (lo + hi) / 2;
            if (a[mid] <= a[hi]) hi = mid;
            else lo = mid + 1;
        }
        return lo;
    }
}
Time: O(log n)  |  Space: O(1)

10. Longest Valid Parentheses Hard

Stack
Intuition: Stack holds indices of unmatched chars. Push -1 as boundary. When ) matches, pop the (. Then i - stack.peek() = length of current valid sequence (top of stack = index just before valid part starts). 
class Solution {
    public int longestValidParentheses(String s) {
        LinkedList<Integer> stack = new LinkedList<>();
        stack.push(-1);
        int res = 0;
        for (int i = 0; i < s.length(); i++) {
            if (s.charAt(i) == ')' && stack.size() > 1
                && s.charAt(stack.peek()) == '(') {
                stack.pop();
                res = Math.max(res, i - stack.peek());
            } else stack.push(i);
        }
        return res;
    }
}
Time: O(n)  |  Space: O(n)

11. Sudoku Solver Hard

Backtracking
Intuition: Find empty cell, try 1-9, check safety, recurse. Box corner formula: r - r%3, c - c%3. Row/col check needs only ONE loop, not nested. 
class Solution {
    public void solveSudoku(char[][] board) { solve(board); }

    private boolean solve(char[][] b) {
        int r = -1, c = -1; boolean found = false;
        for (int i=0; i<9 && !found; i++)
            for (int j=0; j<9 && !found; j++)
                if (b[i][j]=='.') { r=i; c=j; found=true; }
        if (!found) return true;
        for (char x='1'; x<='9'; x++) {
            if (isSafe(b,r,c,x)) {
                b[r][c]=x;
                if (solve(b)) return true;
                b[r][c]='.';
            }
        }
        return false;
    }
    private boolean isSafe(char[][] b, int r, int c, char x) {
        for (int i=0; i<9; i++)
            if (b[r][i]==x || b[i][c]==x) return false;
        int nr=r-r%3, nc=c-c%3;
        for (int i=nr; i3; i++)
            for (int j=nc; j3; j++)
                if (b[i][j]==x) return false;
        return true;
    }
}
Time: O(9^(empty cells))  |  Space: O(81)

12. Find Duplicate Number (Floyd's Cycle)

Linked List Cycle Tortoise & Hare
Intuition: Treat array as linked list: index → nums[index]. Duplicate = cycle. Phase 1: slow + fast meet inside cycle. Phase 2: reset one to 0, both move 1 step → meet at cycle entry = duplicate. 
class Solution {
    public int findDuplicate(int[] nums) {
        int slow = nums[0], fast = nums[nums[0]];
        while (slow != fast) {
            slow = nums[slow];
            fast = nums[nums[fast]];
        }
        fast = 0;
        while (slow != fast) {
            slow = nums[slow];
            fast = nums[fast];
        }
        return slow;
    }
}
Time: O(n)  |  Space: O(1)

13. Permutations Medium

Backtracking
Intuition: Try all unused elements at each position. Use boolean[] used for O(1) lookup instead of list.contains() which is O(n). 
class Solution {
    public List<List<Integer>> permute(int[] nums) {
        List<List<Integer>> res = new ArrayList<>();
        solve(nums, res, new ArrayList<>(), new boolean[nums.length]);
        return res;
    }
    private void solve(int[] a, List<List<Integer>> res,
                       List<Integer> cur, boolean[] used) {
        if (cur.size() == a.length) {
            res.add(new ArrayList<>(cur)); return;
        }
        for (int i = 0; i < a.length; i++) {
            if (used[i]) continue;
            used[i] = true;
            cur.add(a[i]);
            solve(a, res, cur, used);
            cur.remove(cur.size()-1);
            used[i] = false;
        }
    }
}
Time: O(n! × n)  |  Space: O(n)

16. Container With Most Water Medium

Two Pointers
Intuition: Start from both ends. Always move the shorter side inward — moving the taller side can never increase area.
class Solution {
    public int maxArea(int[] h) {
        int i=0, j=h.length-1, max=0;
        while (i < j) {
            max = Math.max(max, Math.min(h[i],h[j]) * (j-i));
            if (h[i] < h[j]) i++; else j--;
        }
        return max;
    }
}
Time: O(n)  |  Space: O(1)

20. Remove Nth Node From End

Linked List Two Pointers
Intuition: Dummy node + move fast n steps ahead, then both until fast hits end. Slow is before target node.
class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {
        ListNode dummy = new ListNode(0, head);
        ListNode fast = dummy, slow = dummy;
        for (int i=0; i<=n; i++) fast = fast.next;
        while (fast != null) { fast=fast.next; slow=slow.next; }
        slow.next = slow.next.next;
        return dummy.next;
    }
}
Time: O(L) one pass  |  Space: O(1)

21. Kth Largest Element

Heap Quickselect
Heap: Min-heap of size k. Top = kth largest. Quickselect: Partition, recurse only into side with target index n-k.

Heap Approach

26. Merge K Sorted Lists Hard

Intuition: Split in half, merge recursively. Remaining nodes attach automatically with curr.next = a.

27. Min Primes to Sum to N (Coin Change)

DP Unbounded Knapsack
Intuition: Generate first m primes. dp[i] = min primes to sum to i. For each value, try every prime. 1D DP is optimal.
class Solution {
    public int minPrimes(int n, int m) {
        List<Integer> primes = genPrimes(m);
        int[] dp = new int[n+1];
        Arrays.fill(dp, Integer.MAX_VALUE);
        dp[0] = 0;
        for (int i=1; i<=n; i++)
            for (int p : primes) {
                if (p>i) break;
                if (dp[i-p] != Integer.MAX_VALUE)
                    dp[i] = Math.min(dp[i], dp[i-p]+1);
            }
        return dp[n]==Integer.MAX_VALUE ? -1 : dp[n];
    }
    private List<Integer> genPrimes(int m) {
        List<Integer> p = new ArrayList<>();
        int num = 2;
        while (p.size() < m) { if(isPrime(num)) p.add(num); num++; }
        return p;
    }
    private boolean isPrime(int n) {
        if(n<2) return false;
        for(int i=2;i*i<=n;i++) if(n%i==0) return false;
        return true;
    }
}
